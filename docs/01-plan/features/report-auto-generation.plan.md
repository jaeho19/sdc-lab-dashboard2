# Plan: report-auto-generation

## Feature Name
월간/주간보고 출력 자동화

## Description
추진경과 데이터를 기간별로 자동 수집하여 보고서 양식 템플릿에 맞춰 자동으로 채워주는 기능.
보고서 자동 생성, 발표 모드, 인쇄 출력, 개인 월간보고 자동 채움을 포함한다.

## Problem Statement

### 사용자 체감 문제
- 매주/매월 보고서 작성 시 추진경과 데이터를 수동으로 복사·붙여넣기해야 함
- 보고서 양식의 업무구분별로 추진경과를 일일이 분류하는 반복 작업
- 회의 시 보고서를 프로젝터에 띄우기 어려움 (별도 PPT 작성 필요)
- 인쇄 시 레이아웃이 양식에 맞지 않아 수동 조정 필요
- 개인 월간 업무보고 작성에도 동일한 반복 작업 발생

### 근본 원인 분석

| # | 원인 | 영향도 | 세부 내용 |
|---|------|:------:|----------|
| 1 | **추진경과→보고서 연결 부재** | Critical | 추진경과 데이터가 있지만 보고서로 자동 연결되는 기능 없음 |
| 2 | **단위사업↔업무구분 매핑 미활용** | High | 단위사업 구조(26121, 26122 등)와 보고서 양식 행(업무구분)이 1:1 매핑 가능하나 활용 안 됨 |
| 3 | **발표용 뷰 부재** | Medium | 보고서 상세 페이지가 일반 웹 레이아웃이라 프로젝터 사용 불편 |
| 4 | **인쇄 레이아웃 미최적화** | Medium | @media print CSS 미적용, A4 양식에 맞지 않음 |

## Solution Overview

### 핵심 개념
"추진경과 데이터 → 보고서 자동 채움"

추진경과 등록 시 이미 입력되는 **프로젝트, 단위사업, 날짜, 제목, 설명, 담당자, 유형** 데이터를
기간별로 자동 수집해서 기존 양식 템플릿에 맞춰 채워준다.

### 구현 단계

#### 1단계: 단위사업↔업무구분 매핑 강화
- 추진경과 등록 시 선택하는 "단위사업"이 보고서 양식의 "행(업무구분)"과 1:1 매핑
- 기존 프로젝트별 단위사업 구조 활용 (26121 거버넌스 역량강화, 26122 돌봄공급자 간 협업체계 구축 등)
- 추진경과가 어느 업무구분 행에 들어갈지 자동 결정

#### 2단계: 보고서 자동 생성 기능
- 보고서 관리 페이지에 "보고서 자동 생성" 버튼 추가
- 자동 생성 다이얼로그:
  - 양식 선택, 프로젝트 선택, 보고 기간 설정
  - 추진경과 자동 수집 미리보기 (단위사업/업무구분별 분류)
  - 체크박스로 포함/제외 선택
  - 차주 계획 자동 반영 (상태가 '계획'인 항목)
  - 보고/회의 유형 자동 포함
- 생성 후 편집 화면으로 이동하여 검토·수정 가능

#### 3단계: 발표 모드 (프레젠테이션)
- 보고서 상세 페이지에 "발표 모드" 버튼 추가
- 전체화면, 프로젝터/큰 스크린 최적화
- 업무구분 2~3개씩 슬라이드 형태로 페이지네이션
- 좌우 화살표 키 / [이전][다음] 버튼 네비게이션
- 글씨 크기 자동 확대
- ESC로 종료

#### 4단계: 인쇄 출력 개선
- 인쇄 설정 다이얼로그 (출력 양식, 용지 방향, 포함 항목 선택)
- CSS @media print 적용: A4 최적화, page-break-inside: avoid
- 기관 로고, 문서 제목, 작성일 자동 삽입
- 사이드바/네비게이션 숨김

#### 5단계: 개인 월간 업무보고 자동 채움
- 담당자 기준으로 해당 월 추진경과 자동 수집
- 프로젝트별 분류 → 양식 섹션별 자동 매핑:
  - 섹션 2 "월간 주요 업무 추진실적" → 프로젝트별 행 자동 생성
  - 섹션 3 "핵심 과업" → 완료 상태 추진경과
  - 섹션 3 "협업 및 대외 업무" → '회의' 유형 추진경과
  - 섹션 5 "차월 추진계획" → '계획' 상태 추진경과

## Implementation Phases

- [ ] 단위사업-보고서 양식 행 매핑 테이블/로직 구현
- [ ] 추진경과 기간별 조회 API (Supabase query)
- [ ] 담당자별 추진경과 조회 API

### Phase 2: 보고서 자동 생성 UI
- [ ] AutoGenerateReportDialog 컴포넌트
- [ ] 양식 선택 → 연결 프로젝트 자동 필터
- [ ] 기간 설정 (주간/월간 자동 감지)
- [ ] 추진경과 미리보기 (업무구분별 트리 + 체크박스)
- [ ] 보고서 생성 로직 (체크된 항목 → 보고서 sections 자동 채움)

### Phase 3: 발표 모드
- [ ] PresentationMode 컴포넌트 (전체화면, 슬라이드 페이지네이션)
- [ ] 키보드 네비게이션 (Arrow Left/Right, ESC)
- [ ] 업무구분 기준 슬라이드 분할 로직
- [ ] 반응형 글씨 크기

### Phase 4: 인쇄 출력
- [ ] PrintDialog 컴포넌트 (출력 양식, 용지 방향, 포함 항목)
- [ ] @media print CSS (A4 최적화, page-break 제어)
- [ ] 기관 로고/헤더/푸터 인쇄 전용 레이아웃

### Phase 5: 개인 월간보고 자동 채움
- [ ] 담당자 기준 추진경과 수집 로직
- [ ] 개인 월간 양식 섹션별 자동 매핑 로직
- [ ] 월간보고 전용 AutoGenerateDialog 변형

## Technical Considerations

### 데이터 의존성
- `progress_logs` 테이블: 추진경과 데이터 (project_id, sub_project_id, date, title, description, assignee, type, status)
- `report_templates` 테이블: 보고서 양식 정의 (sections, rows/columns 구조)
- `sub_projects` 테이블: 단위사업 목록 (code, name, project_id)
- `reports` 테이블: 생성된 보고서 저장

### 성능 고려
- 기간별 추진경과 조회 시 인덱스 활용 (date, project_id)
- 미리보기 데이터는 클라이언트 상태로 관리 (서버 왕복 최소화)
- 발표 모드는 CSS transform 기반 (DOM 재생성 없이 슬라이드 전환)

### UX 핵심 원칙
- 추진경과만 성실히 등록하면 보고서 작성 시간이 "체크박스 몇 번 + 내용 보완"으로 축소
- 자동 생성 후 반드시 편집 가능 (완전 자동이 아닌 반자동)
- 기존 수동 보고서 작성 워크플로우도 그대로 유지

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| 단위사업↔업무구분 매핑이 양식마다 다를 수 있음 | High | 양식별 매핑 설정 UI 제공 |
| 추진경과 데이터 품질이 낮으면 자동 생성 결과도 저품질 | Medium | 미리보기에서 확인 + 편집 가능하게 |
| 발표 모드에서 콘텐츠가 슬라이드에 안 맞을 수 있음 | Medium | 동적 슬라이드 분할 + overflow 처리 |
| 인쇄 레이아웃이 브라우저마다 다르게 렌더링 | Low | Chrome/Edge 기준 최적화, 테스트 |

## Success Metrics
- 주간 보고서 작성 시간: 30분 → 5분 이내
- 월간 개인 보고서 작성 시간: 1시간 → 10분 이내
- 보고서 자동 생성 사용률: 80% 이상 (수동 작성 대비)
- 발표 모드 활용: 주간 회의 시 100% 사용

## Priority
**High** - 반복적인 보고서 작성 업무가 가장 큰 시간 낭비 요소
